# Публикация DocSScaner в RuStore

## Что уже настроено

- Package name и namespace: `ru.alkatrazer.docscaner`.
- `minSdk 26`, `targetSdk 36`, `versionCode 90`, `versionName 1.0`.
- Release-сборка требует отдельный ключ и не создаётся неподписанной по ошибке.
- APK/AAB не разрешает незашифрованный HTTP-трафик.
- Сканам, OCR-данным, журналам и настройкам запрещено облачное резервное копирование и перенос средствами Android.
- Иконка каталога: `metadata/en-US/images/icon.png`, 512×512 PNG.
- Русские тексты карточки: `metadata/ru/`.
- Политика конфиденциальности: `docs/PRIVACY_POLICY_RU.md`.

## 1. Действия владельца приложения

1. Написание бренда и package name подтверждено владельцем: `DocSScaner` и `ru.alkatrazer.docscaner`. Package name нельзя менять после первой публикации.
2. Опубликуйте исходный код именно этой изменённой версии в доступном репозитории под GPLv3. Добавьте ссылку на репозиторий в экран «О приложении» и в карточку RuStore. Ссылка только на исходный FairScan не заменяет публикацию исходников вашей сборки.
3. Опубликуйте `docs/PRIVACY_POLICY_RU.md` по постоянному HTTPS-адресу и укажите этот адрес в RuStore. Перед публикацией проверьте контактный email.
4. Зарегистрируйте аккаунт разработчика в RuStore Консоли. Для карточки выберите категорию «Инструменты» и возрастную категорию `0+`.

## 2. Создание ключа подписи

Храните ключ вне репозитория и сделайте зашифрованную резервную копию. Потеря ключа осложнит выпуск обновлений.

### Простой способ

Запустите двойным щелчком `СОБРАТЬ_ДЛЯ_RUSTORE.bat`. При первом запуске программа попросит придумать пароль не короче 12 символов, создаст ключ в папке `Документы\DocSScaner-release` и соберёт файл `release\DocSScaner-RuStore.apk`.

Пароль не отображается при вводе. Не удаляйте папку `Документы\DocSScaner-release`: ключ потребуется для всех будущих обновлений приложения. Сделайте резервную копию этой папки на защищённый носитель.

### Ручной способ

Пример команды из JDK:

```powershell
keytool -genkeypair -v -storetype PKCS12 -keystore C:\secure\DocSScaner-release.jks -alias docsscaner -keyalg RSA -keysize 4096 -validity 10000
```

Скопируйте `release-signing.properties.example` в `release-signing.properties` и заполните настоящий путь, alias и пароли. Этот файл и файлы ключей исключены из Git.

Параметры также можно передать через переменные окружения или Gradle properties:

- `RELEASE_STORE_FILE`
- `RELEASE_STORE_PASSWORD`
- `RELEASE_KEY_ALIAS`
- `RELEASE_KEY_PASSWORD`

## 3. Сборка APK

Для Gradle требуется JDK 17 или совместимая JDK, поддерживаемая используемой версией Android Gradle Plugin.

```powershell
.\gradlew.bat --no-daemon test :app:lintRelease :app:assembleRelease
```

Файл для загрузки появится по адресу:

```text
app/build/outputs/apk/release/DocSScaner-1.0-universal.apk
```

Для проверки сертификата используйте `apksigner` из Android SDK:

```powershell
apksigner verify --verbose --print-certs release\DocSScaner-RuStore.apk
```

Проект создаёт один универсальный APK, а также отдельные APK для `arm64-v8a`, `armeabi-v7a` и `x86_64`. В RuStore загружайте только `release\DocSScaner-RuStore.apk`: его подпись уже находится внутри, поэтому отдельная настройка подписи AAB не требуется.

## 4. Декларация безопасности данных

Укажите в RuStore:

- камера: обязательна для основной функции сканирования;
- изображения и файлы: обрабатываются локально, не собираются разработчиком и передаются только по действию пользователя;
- интернет: используется для загрузки языковых моделей OCR с `raw.githubusercontent.com`;
- хранилище: разрешение действует только до Android 9 включительно и используется для сохранения PDF;
- реклама, аналитика, аккаунты, платежи и фоновые трекеры отсутствуют.

Перед отправкой проверьте эти ответы по фактическому поведению итоговой сборки.

## 5. Карточка приложения

- Название: `DocSScaner – Сканер в PDF` (до 30 символов).
- Категория: «Инструменты».
- Возраст: `0+`.
- Контакт: `profcapper@gmail.com`.
- Краткое описание: `metadata/ru/short_description.txt` (до 80 символов).
- Полное описание: `metadata/ru/full_description.txt` (до 4000 символов).
- Что нового: `metadata/ru/changelogs/90.txt`.
- Иконка: квадратная 512×512, без прозрачных краёв, визуально такая же, как иконка установленного приложения.
- Скриншоты: от 1 до 10, PNG/JPG, русский или английский интерфейс, 9:16 либо 16:9, без статус-бара, навигационной панели и интерфейса других приложений.

Текущие четыре скриншота показывают реальный интерфейс, но содержат системные панели Android и имеют неподходящую пропорцию. Перед модерацией сделайте новые скриншоты итоговой release-сборки в пропорции 9:16 без системных элементов.

## 6. Финальная проверка

1. Установите подписанный release APK как чистую установку.
2. Проверьте запрос камеры, съёмку, импорт, обрезку, OCR, загрузку языка, сохранение и отправку PDF.
3. Проверьте запуск через `GET_CONTENT` и action `ru.alkatrazer.docscaner.action.SCAN_TO_PDF`.
4. Убедитесь, что приложение не падает без сети и при отказе в разрешении камеры.
5. Проверьте отображение имени и иконки на русском устройстве.
6. Сохраните ключ, alias, пароли и SHA-256 сертификата в защищённом хранилище.
7. Загрузите `release\DocSScaner-RuStore.apk` в RuStore, заполните декларацию разрешений, добавьте карточку и отправьте на модерацию.
